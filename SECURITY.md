# Security Policy

## Supported Versions

Security fixes are provided on a best-effort basis for the latest version on the default branch.

## Reporting a Vulnerability

Please do not open a public issue for suspected security vulnerabilities.

Email the details to the project maintainer or use a private GitHub security advisory if available. Include:

- a description of the issue
- steps to reproduce it
- the affected version or commit
- any suggested mitigations

I will acknowledge receipt as soon as practical and coordinate a fix before public disclosure when possible.
